# Perth Mint: как устроены URL картинок монет

По разметке сайта Perth Mint картинки монет лежат по единому правилу. Это можно использовать, чтобы забирать изображения без парсинга каждой страницы (если известны год и SKU).

## Паттерн URL

```
https://www.perthmint.com/globalassets/assets/product-images-e-com-pages/coins/{YEAR}/{SKU}/{FILENAME}.jpg
```

- **YEAR** — год выпуска в пути, например `2026`, `2025`.
- **SKU** — идентификатор продукта в нижнем регистре, например `26y15aaa` (на странице указан как `26Y15AAA`).
- **FILENAME** — имя файла: порядковый номер, описание монеты, тип вида. Примеры:
  - `01-...-rev-left.jpg`, `02-...-rev.jpg` — реверс
  - `03-...-obverse.jpg` — аверс
  - `04-...-box-front.jpg` — упаковка

Пример полного URL:

`/globalassets/assets/product-images-e-com-pages/coins/2026/26y15aaa/02-deadly-dangerous-2026-giant-centipede-1oz-silver-proof-coin-rev.jpg?contextmode=Default?width=500`

Параметр **`width=500`** даёт уменьшенную версию; без него или с `width=2000` сервер отдаёт изображение в полном разрешении (до 2000×2000).

## Как искать другие монеты

1. **Где взять YEAR и SKU**
   - Из URL страницы товара (если там есть год или SKU).
   - Из HTML страницы товара (атрибуты `data-*`, мета-теги, текст «SKU: …»).
   - Со списка/каталога Perth Mint (если получится его выгрузить или спарсить).

2. **Как получить имена файлов**
   - **Надёжно:** один раз открыть страницу товара (Playwright/браузер), собрать все `img[src]` из блока галереи — там будут точные имена файлов для этого SKU.
   - **Эвристика:** для известного SKU можно пробовать типовые шаблоны вроде `01-*-obv.jpg`, `02-*-rev.jpg` и т.д., но точное имя (описание монеты в имени файла) без страницы не узнать.

3. **Прямая загрузка по URL**
   - Если год + SKU + имя файла уже известны (например, из предыдущего парсинга страницы), картинку можно качать обычным `fetch()` по полному URL, без браузера — аналогично тому, как мы качаем картинки ЦБ по каталожному номеру.

## Каталог монет Perth Mint (откуда брать список)

В отличие от ЦБ, у Perth Mint **нет готового каталога в Excel/CSV**. Варианты:

1. **Официальный сайт**
   - Раздел каталога (PDF): [Latest catalogue](https://www.perthmint.com/collector-coins/information-for-coin-collectors/latest-catalogue/) — это буклет в PDF, не таблица.
   - **Магазин (список товаров):**  
     `https://www.perthmint.com/shop/collector-coins/?archived=false&page=1&pageSize=12&sortValue=0`  
     Страница с пагинацией (`page=1`, `pageSize=12`). Можно открыть в браузере (Playwright), собрать ссылки на карточки товаров — получится список URL страниц монет. По каждому URL потом запускать `fetch-perth-mint-coin.js`.
   - Тиражы по сериям иногда в PDF: [Declared mintages](https://www.perthmint.com/collector-coins/information-for-coin-collectors/declared-mintages/).

2. **Colnect**  
   [Каталог Perth Mint на Colnect](https://colnect.com/en/coins/series/mint/5-Perth_Mint_Perth_Australia) — сторонний каталог по сериям/годам. Можно парсить для списка названий и годов; SKU и точные URL картинок всё равно брать с сайта Perth Mint.

3. **Итог**  
   Удобной единой страницы «все монеты в таблице» или Excel, как у ЦБ, у Perth Mint нет. Реалистичный вариант: парсить страницы магазина (список товаров с пагинацией), собирать URL товаров, затем для каждого URL вызывать текущий скрипт за данными и картинками.

## Скрипты в проекте

- **`scripts/fetch-perth-mint-coin.js`** — открывает одну страницу товара, вытаскивает данные и URL картинок, подставляет `width=2000` для максимального разрешения, сохраняет в `public/image/coins/foreign/` и данные в `data/perth-mint-*.json`.
- Для массового сбора: скрипт, который открывает `shop/collector-coins/?page=1&pageSize=24`, собирает ссылки на товары со страницы (и при необходимости обходит страницы по `page=2,3,...`), затем для каждого URL вызывает `fetch-perth-mint-coin.js` с паузой между запросами.
