# Полная синхронизация Perth Mint: два этапа

Цель: все монеты Perth на сайте с актуальными данными и изображениями, **без объединения** (1 ссылка = 1 монета), **без устаревшего кэша**.

---

## Как ты сформулировал — верно

1. **Этап 1.** Проверить ссылки и забрать себе: страницы монет со спецификацией + изображения (всё заново, без кэша).
2. **Этап 2.** Импорт и экспорт: чтобы монеты оказались на сайте с правильной информацией и изображениями, каждая монета отдельно.

Порядок правильный: сначала забираем данные и картинки, потом заливаем в БД и на сайт.

---

## Этап 1: Проверка ссылок и забор страниц + изображений

- **Список URL:** `scripts/perth-mint-urls.txt` (или свой файл).
- **Проверка ссылок:** при запуске fetch каждая ссылка открывается; если страница не загружается — будет ошибка в консоли (битые ссылки можно потом выкинуть из списка).
- **Забор без кэша (полный перезабор):**
  ```bash
  cd /path/to/omonete-app
  node scripts/fetch-perth-mint-coin.js --refresh --no-image-cache
  ```
  - `--refresh` — обрабатываются **все** URL из списка, в том числе уже «завершённые» (перезапись JSON и картинок).
  - `--no-image-cache` — кэш картинок не используется: все изображения качаются заново и сохраняются в `public/image/coins/foreign/`.
- **Результат:** в `data/` — JSON по каждой монете (спеки, страна, номинал, вес 7.78 и т.д.), в `public/image/coins/foreign/` — актуальные obverse/reverse/box/certificate.

Если нужно обработать только «потерянные» URL (из отдельного списка):

```bash
node scripts/list-perth-mint-refetch-urls.js   # сформировать scripts/perth-mint-refetch-urls.txt
node scripts/fetch-perth-mint-coin.js --refetch-lost --refresh
```

---

## Этап 2: Монеты на сайте (импорт + экспорт)

- **Импорт в БД** (из JSON в `data/perth-mint-*.json`):
  ```bash
  node scripts/import-perth-mint-to-db.js --refresh
  ```
  `--refresh` обновляет существующие записи по `catalog_number` (или по title+mint), так что в БД попадут актуальные country, face_value, weight_g (7.78), length_mm/width_mm, изображения и т.д.

- **Экспорт на сайт** (БД → `public/data/coins.json` и отдельные JSON страниц монет):
  ```bash
  node scripts/export-coins-to-json.js
  ```
  После этого на сайте будут отображаться все монеты **без объединения** (одна запись в БД = одна монета на сайте), с правильными полями и путями к изображениям.

---

## Краткий чеклист

1. Обновить список URL в `scripts/perth-mint-urls.txt` (если нужно).
2. Запустить **Этап 1:** `node scripts/fetch-perth-mint-coin.js --refresh --no-image-cache`.
3. Проверить выборочно JSON в `data/` и картинки в `public/image/coins/foreign/`.
4. Запустить **Этап 2:** `node scripts/import-perth-mint-to-db.js --refresh`, затем `node scripts/export-coins-to-json.js`.
5. Проверить каталог и страницы монет на сайте.

Готово: данные только из страниц Perth, без хардкода; веса 7.78/15.55 не округляются; страна и номинал из спеок; прямоугольные по length/width; изображения без использования старого кэша при полном перезаборе.

---

## Пропуск монет и понимание, кто «нормально» забран

- **Без `--refresh`:** скрипт **пропускает** все URL из `data/perth-mint-fetch-progress.json` (считаем их уже обработанными). Обрабатываются только те ссылки, которых ещё нет в `completedUrls`. То есть «нормально спарсившиеся» монеты при следующем запуске по умолчанию пропускаются.
- **С `--refresh`:** никого не пропускаем — все URL из списка обрабатываются заново (перезапись JSON и картинок).

**Кто куда попадает:**
- В прогресс (`completedUrls` + `coins`) попадают только те URL, по которым fetch **успешно завершился** (страница загрузилась, данные извлечены, JSON записан). При ошибке (таймаут, падение) URL в прогресс **не** добавляется — такие ссылки при следующем запуске без `--refresh` снова пойдут в обработку.
- В каждой записи в `coins` теперь есть поле **`status`**: `"ok"` (есть хотя бы аверс или реверс) или `"partial"` (спеки есть, но основных картинок нет).

**Как понять, у кого дубликаты или проблемы:**
- **Отчёт по прогрессу и полноте данных:**  
  `node scripts/report-perth-mint-status.js`  
  Показывает: сколько в прогрессе, сколько «OK» (есть спеки и картинки), у кого нет JSON, нет спеок, нет картинок, сколько URL в списке не в прогрессе (ещё не брали или ошибка при fetch).
- **Дубликаты изображений** (одинаковые файлы под разными именами):  
  `node scripts/dedupe-perth-mint-images.js --dry-run` — только отчёт; без `--dry-run` — исправление JSON и удаление лишних файлов.
