# Тарифы и монетизация

## Идея

- **Бесплатный тариф** — базовые возможности для всех.
- **Платные тарифы** — расширенные лимиты и функции для монетизации проекта.

## Варианты структуры

### 1. По размеру коллекции

| Тариф        | Лимит монет в портфолио | Цена          |
|-------------|--------------------------|---------------|
| Бесплатный  | до 50 монет              | 0 ₽           |
| Базовый     | до 500 монет             | подписка/год  |
| Про         | без лимита               | подписка/год  |

### 2. По функциям

- **Бесплатно:** каталог, графики, портфолио с лимитом (например 50 монет), просмотр цен.
- **Платно:** снятие лимита, экспорт в Excel, напоминания, приоритетная поддержка, возможно отдельные отчёты/аналитика.

### 3. Разово + подписка

- Разовый платёж за «расширенное портфолио» на год.
- Или ежемесячная/годовая подписка с автоматическим продлением.

## Что нужно для внедрения

1. **Бэкенд:** хранение тарифа пользователя (Supabase: поле в `profiles` или отдельная таблица `subscriptions`).
2. **Проверка лимитов:** при добавлении в коллекцию и при отображении портфолио проверять лимит по тарифу.
3. **Платежи:** интеграция платёжного провайдера (ЮKassa, Stripe, Tinkoff и т.п.) для приёма оплаты и выдачи платного тарифа.
4. **Страница тарифов:** блок или отдельная страница «Тарифы» с описанием и кнопкой оплаты.

## Дальнейшие шаги

- Выбрать модель (лимит по количеству / по функциям).
- Добавить поле тарифа в профиль пользователя.
- Реализовать проверку лимита в портфолио и при добавлении монеты.
- Подключить один платёжный сервис и тестовый сценарий оплаты.
- Добавить страницу или секцию «Тарифы» в приложение.
