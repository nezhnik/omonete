# Тестовая БД для иностранных монет

## Архитектура: одна БД для всех монет

**Рекомендация:** использовать одну БД и одну таблицу `coins` для российских и иностранных монет.

| Критерий | Значение |
|----------|----------|
| Различие | Поле `country` (Россия, США, Австралия и т.д.) |
| Фильтр в каталоге | Вкладки «Все» / «Российские» / «Иностранные» по `country` |
| Поиск | По `title`, `title_en`, `series` и др. |

**Почему не две отдельные БД:**
- Усложняет каталог (два источника, два API)
- Усложняет поиск и фильтры
- Дублирование логики (экспорт, синхронизация)

## Тестовая БД для разработки

Для безопасной проверки импорта иностранных монет можно использовать **отдельную тестовую БД**.

### Вариант 1: `DATABASE_URL_FOREIGN` (импорт-скрипты)

Если задана переменная `DATABASE_URL_FOREIGN`, скрипты импорта иностранных монет пишут в неё, а не в основную БД.

```env
# .env (не коммитить — добавить в .gitignore)
DATABASE_URL="mysql://user:pass@host:3310/prod_db"
DATABASE_URL_FOREIGN="mysql://user:pass@host:3310/omonete_foreign_test"
```

Скрипт импорта (когда будет создан):

```js
const dbUrl = process.env.DATABASE_URL_FOREIGN || process.env.DATABASE_URL;
```

### Вариант 2: Отдельная БД на Reg.ru

1. В панели Reg.ru создайте вторую БД (например `u2524954_omonete_test`)
2. Скопируйте схему: `create-coins-table.sql` + миграции
3. Добавьте `title_en`: `node scripts/ensure-title-en-column.js` (предварительно подставьте `DATABASE_URL_FOREIGN` в `.env`)
4. Импортируйте иностранные монеты в тестовую БД
5. Проверьте каталог (переключите `DATABASE_URL` на тестовую)
6. При успехе — импорт в прод (основную БД)

### Миграция title_en

Перед импортом иностранных монет нужно добавить столбец `title_en`:

```bash
# Автоматически (если колонки нет)
node scripts/ensure-title-en-column.js

# Или вручную
mysql -u user -p database < scripts/add-title-en-column.sql
```

## Итог

| Вопрос | Ответ |
|-------|-------|
| Одна БД для RU и иностранных? | Да, одна таблица `coins`, различие по `country` |
| Тестовая БД для иностранных? | Да, для разработки/импорта — опционально `DATABASE_URL_FOREIGN` |
| Когда использовать две БД? | Только для тестирования импорта, не для продакшена |
