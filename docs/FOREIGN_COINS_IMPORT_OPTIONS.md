# Варианты забора иностранных монет с изображениями

Цель: начать забирать к нам монеты (данные + картинки) и выкладывать в каталог. Ниже — варианты по источникам и объёму работ.

---

## Общий пайплайн (как у нас уже устроено)

1. **Список монет** — откуда-то получить (CSV, API, парсинг списка страниц).
2. **Метаданные** — страна, год, номинал, металл, вес, тираж, название; при необходимости маппинг в нашу схему БД.
3. **Изображения** — скачать с источника или официального сайта, конвертировать в WebP, положить в `public/image/coins/foreign/`.
4. **БД** — вставка/обновление в `coins` (как ASE, Perth одна монета).
5. **Экспорт** — `export-coins-to-json.js` уже выгружает всё в статику; монеты с ненулевым тиражом попадают в каталог.

---

## Вариант 1: Perth Mint — масштабировать текущий скрипт

**Что есть:** `fetch-perth-mint-coin.js` забирает одну страницу товара Perth Mint (данные + картинки). Список монет можно взять с Colnect (CSV до 100 шт. или API) или парсить каталог магазина Perth Mint.

**Что сделать:**
- Скрипт **списка URL** страниц Perth Mint: открыть каталог магазина (пагинация `page=1&pageSize=12`), собрать ссылки на товары, сохранить в JSON/CSV.
- Запускать `fetch-perth-mint-coin.js` по каждой ссылке с паузой (например 3–5 с), чтобы не нагружать сайт.
- По каждой сохранённой монете (JSON + картинки) — скрипт импорта в БД (аналог `import-ase-to-db.js`): читаем наш JSON, вставляем в `coins`, проставляем `image_obverse`/`image_reverse` на скачанные файлы.

**Плюсы:** один источник (официальный сайт), хорошие картинки, данные и картинки в одном месте.  
**Минусы:** только Perth Mint; парсинг постраничный, нужны паузы.  
**Объём:** 300+ нумизматических + 20 буллонных — реально за несколько дней при аккуратном парсинге.

## Вариант 2: Colnect — список + картинки с Colnect (по API или экспорту)

**Что есть:** Colnect API (по заявке, бесплатно для некоммерческого) или экспорт CSV (до 100 бесплатно, до 10k Premium). На карточках монет есть изображения (часто от пользователей).

**Что сделать:**
- Подать заявку на Colnect API, получить ключ.
- По API запрашивать список монет по стране или монетному двору (например Австралия, или mint=Perth Mint). В ответах смотреть, есть ли URL изображений и условия использования.
- Скрипт: для каждой монеты из API — сохранить метаданные, скачать изображения (если в API отдаются URL и лицензия позволяет), конвертировать в WebP, импорт в нашу БД.
- Важно: проверить **правила Colnect** по использованию изображений (авторские права, коммерческое/некоммерческое). Если картинки нельзя переиспользовать — использовать Colnect только как **список** (название, год, номинал и т.д.), а картинки брать с официальных сайтов дворов (Вариант 1, 3).

**Плюсы:** большой каталог по странам/дворам, один API.  
**Минусы:** лицензия картинок под вопросом; защита от ботов на сайте (парсинг страниц сложнее, API предпочтительнее).

## Вариант 3: Официальные сайты дворов (кроме Perth)

**Источники:**
- **US Mint** (catalog.usmint.gov) — каталог товаров, можно парсить списки и страницы товаров (данные + изображения). У нас уже есть ASE из других источников; можно добавить другие линейки (America the Beautiful и т.д.).
- **Royal Canadian Mint** — магазин/каталог, обычно есть списки продуктов и изображения.
- **Royal Mint (UK)** — аналогично.
- **Прочие дворы** — по мере необходимости (Австрия, Германия и т.д.), если есть каталог с URL товаров.

**Что сделать:**
- Для выбранного двора: скрипт обхода каталога (если есть пагинация/API) → сбор URL страниц товаров.
- Для каждой страницы: парсинг (Playwright/Puppeteer, если нужен JS) — название, год, номинал, металл, вес, тираж, URL картинок; скачивание картинок, WebP, сохранение.
- Импорт в БД по той же схеме, что и для ASE/Perth (страна, catalog_number или внешний ID, image_obverse/reverse).

**Плюсы:** официальные данные и картинки, меньше вопросов с правами.  
**Минусы:** разный формат у каждого двора, под каждый нужен свой парсер.

## Вариант 4: Numista API — данные и (если есть) изображения

**Что есть:** [Numista API](https://en.numista.com/api/) — поиск монет, детали по типу, возможно изображения. Нужна регистрация и ключ; условия и лимиты смотреть на сайте.

**Что сделать:**
- Изучить документацию API: какие поля отдаются, есть ли URL изображений и лицензия.
- Если изображения отдаются и разрешены к использованию — скрипт: запросы по странам/периодам, сохранение метаданных и картинок, импорт в БД.
- Если изображения в API нет или лицензия не позволяет — использовать Numista только как **источник списка** (название, год, номинал, металл, вес), а картинки искать на официальных сайтах дворов или не использовать.

**Плюсы:** очень большой каталог, единый API.  
**Минусы:** нужно проверить доступ к картинкам и ToS.

## Вариант 5: Платный Excel (CoinsCatalogs.com) + свои картинки

**Что есть:** CoinsCatalogs.com — платные тома в Excel по буквам стран (десятки тысяч монет в томе). В Excel — метаданные (страна, год, номинал и т.д.), но не факт, что там URL картинок.

**Что сделать:**
- Приобрести один–два тома (например «A» — Австралия, Австрия и др.) для пробы.
- Скрипт чтения Excel: парсинг строк, маппинг колонок в нашу схему, вставка в БД с пустыми `image_obverse`/`image_reverse` или заглушкой.
- Картинки потом: вручную для топ-монет или парсинг официальных сайтов по названию/году/номиналу (сложнее автоматизировать).

**Плюсы:** сразу много структурированных данных.  
**Минусы:** платно; изображений в базе нет — их нужно добывать отдельно.

## Вариант 6: Гибрид — Colnect/Numista как справочник, картинки с дворов

**Идея:** Список монет и метаданные брать из Colnect (API/CSV) или Numista (API), а **изображения всегда** — только с официальных сайтов монетных дворов (Perth Mint, US Mint, Royal Canadian Mint и т.д.).

**Что сделать:**
- Colnect/Numista: получаем список (страна, название, год, номинал, вес, тираж где есть).
- По названию/году/номиналу/стране определяем двор (или в Colnect уже есть mint) и ищем страницу товара на сайте двора (вручную для первых партий или простой поиск по URL каталога).
- С страницы двора забираем картинки и при необходимости уточняем данные (тираж и т.д.).
- Импорт в БД: метаданные из Colnect/Numista, пути к картинкам — наши скачанные файлы.

**Плюсы:** богатый список, легальные картинки, меньше рисков по авторским правам.  
**Минусы:** два шага (список + двор); не у всех монет найдётся страница на официальном сайте.

## Рекомендация: с чего начать

1. **Быстрый старт (мало рисков, всё под контролем)**  
   - **Perth Mint (Вариант 1):** дописать скрипт сбора списка URL из каталога магазина, цикл по URL с паузой + текущий `fetch-perth-mint-coin.js`, затем импорт в БД. Так мы сразу получаем и данные, и картинки с одного официального источника.

2. **Расширение списков без картинок**  
   - Оформить **Colnect API** (Вариант 2) и выгружать списки по странам/дворам. Сначала использовать только метаданные; картинки подставлять потом — с Perth Mint, US Mint и т.д. (Вариант 3), либо оставлять плейсхолдер.

3. **Добавить ещё один двор**  
   - Выбрать один двор с удобным каталогом (например **Royal Canadian Mint** или **US Mint** помимо ASE) и сделать под него парсер по образцу Perth Mint: список страниц → парсинг страницы → картинки + данные → импорт в БД.

4. **Проверить Numista (Вариант 4)**  
   - Посмотреть в API, отдаются ли URL изображений и что говорят условия использования. Если да и разрешено — можно пробовать импорт по странам с картинками из Numista.

Итого: логично **начать с масштабирования Perth Mint** (список URL + цикл fetch + импорт в БД), параллельно подать заявку на **Colnect API** и решить, использовать ли их только как каталог или и картинки (после проверки правил). Остальные варианты — по мере необходимости расширять каталог и источники изображений.
