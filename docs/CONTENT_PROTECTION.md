# Защита контента (фото монет) от копирования и парсинга

## Что уже сделано

### 1. Запрет горячих ссылок (`.htaccess`)

В **public/.htaccess** добавлено правило: запросы к папке **/image/** с другого сайта (когда в заголовке Referer чужой домен) получают **403 Forbidden**. Картинки отдаются только:
- при открытии страниц с omonete.ru (Referer = omonete.ru);
- при прямом запросе (Referer пустой — например, новая вкладка).

**Итог:** вставить ваши фото на чужой сайт через `<img src="https://omonete.ru/image/...">` не получится — браузер не покажет картинку. Массово качать по списку URL без Referer по-прежнему возможно, но воровство «под видом своего сайта» блокируется.

После деплоя содержимое `out` попадает в корень сайта, в том числе этот `.htaccess`.

## Что можно добавить по желанию

### 2. Водяной знак на изображениях

Нанести на каждое фото монеты полупрозрачный текст или логотип (например, «omonete.ru»). Тогда даже при копировании изображения источник виден.

Варианты:
- Скрипт на Node (sharp): при сборке или отдельной задаче проходить по файлам в `public/image/coins/` и накладывать текст.
- Делать водяной знак только на превью в каталоге, а на странице монеты показывать без него (сложнее логистика).

### 3. Юридическая подпись

В подвале сайта и/или на странице с правилами указать:

- «Фотографии монет и материалы сайта являются объектами авторских прав. Использование без письменного разрешения правообладателя запрещено.»

При воровстве это даёт основание для претензии и запроса удаления (в т.ч. через хостинг/поисковики).

### 4. Отключение правой кнопки мыши (по желанию)

Можно отключить контекстное меню на страницах с картинками (`onContextMenu` → `preventDefault`). Это лишь усложняет «Сохранить как» для обычного пользователя; парсеры и DevTools это обходят. Многие считают такой приём неудобным для пользователей, поэтому добавлять только если вы осознанно готовы к этому.

## Ограничения

- На статическом сайте (только HTML/JS/картинки) **полностью** закрыть доступ к файлам для того, кто может открыть страницу в браузере, нельзя: картинки уже подгружаются в браузер.
- Защита по Referer и водяной знак **существенно снижают** удобство воровства и использование ваших фото на других сайтах; целеустремлённый человек при ручном сохранении всё равно может сохранить файл.

Рекомендуемая комбинация: **hotlink (уже есть) + текст об авторских правах + при желании водяной знак.**
