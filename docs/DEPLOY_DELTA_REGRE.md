# Инкрементальная выгрузка на Reg.ru (папка regre)

Чтобы не заливать каждый раз все ~800 МБ из `out/`, после сборки можно сформировать папку **`out/regre`** только с теми файлами, которые изменились или появились с прошлого билда.

## Как это устроено

1. **После билда** запускается скрипт `prepare-deploy-regre.js`:
   - Сравнивает текущее содержимое `out/` с манифестом прошлого деплоя (`deploy-manifest.json` в корне проекта).
   - В **`out/regre/`** копирует только **новые** и **изменённые** файлы, сохраняя структуру каталогов (например, `out/regre/coins/1234.html`, `out/regre/data/coins.json`).
   - В **`out/regre/deleted.txt`** записывает пути файлов, которые в этом билде удалены (их нужно удалить на сервере).
   - Обновляет `deploy-manifest.json` для следующего сравнения.

2. **На Reg.ru** заливаете не весь `out/`, а только содержимое **`out/regre/`**:
   - Копируете файлы из `regre/` в корень сайта с теми же путями (например, `regre/coins/1234.html` → `www/omonete.ru/coins/1234.html`).
   - Файлы из `deleted.txt` на сервере удаляете (если используете список удалений).

3. **Первый раз** манифеста нет — скрипт помечает все файлы как «новые», и в `out/regre/` попадает полная копия (как при обычной полной выгрузке). Либо первый деплой делаете как раньше (всё из `out/`), потом уже используете `regre`.

## Команды

```bash
# Сборка как обычно
npm run build

# Сформировать out/regre (только изменённые и новые)
node scripts/prepare-deploy-regre.js
```

Дальше на Reg.ru заливаете только содержимое `out/regre/` (и при необходимости обрабатываете `out/regre/deleted.txt`).

## Где что лежит

| Что | Где |
|-----|-----|
| Полный статический сайт | `out/` |
| Манифест (путь → хэш) для сравнения | `deploy-manifest.json` (корень проекта) |
| Дельта для загрузки | `out/regre/` |
| Список путей к удалению на сервере | `out/regre/deleted.txt` |

## Примечания

- `deploy-manifest.json` не кладите в репозиторий, если не хотите, чтобы дельта считалась от коммита; тогда каждый клон после первого билда будет создавать полную копию в `regre`.
- Если нужна «полная выгрузка с нуля», удалите `deploy-manifest.json` и снова запустите `prepare-deploy-regre.js` — в `regre` попадёт всё содержимое `out/`.
