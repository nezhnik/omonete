# Supabase по шагам (вход через GitHub)

Я не могу войти в Supabase под твоим аккаунтом. Ниже — что делаешь ты; после шагов 1–3 могу добавить в проект код (клиент, формы входа, работа с коллекцией).

---

## Шаг 1. Проект в Supabase (ты)

1. Открой [supabase.com](https://supabase.com).
2. Нажми **Start your project** → войди через **GitHub** (authorize Supabase).
3. **New project**:
   - Organization: оставь дефолтную или создай новую.
   - Name: например `omonete`.
   - Database Password: придумай и **сохрани** (для доступа к БД напрямую, для кода сайта не обязателен).
   - Region: ближайший (например Frankfurt).
4. Нажми **Create new project** и дождись создания (1–2 минуты).

---

## Шаг 2. Создать таблицу коллекции (ты)

1. В левом меню проекта Supabase открой **SQL Editor**.
2. Нажми **New query**.
3. Открой в проекте файл **`supabase/schema.sql`**, скопируй весь его текст и вставь в запрос в SQL Editor.
4. Нажми **Run** (или Ctrl+Enter).
5. Внизу должно быть сообщение об успешном выполнении. Таблица `user_collection` и политики RLS созданы.

Проверка: в меню **Table Editor** должна появиться таблица **user_collection** с колонками `id`, `user_id`, `coin_id`, `added_at`.

---

## Шаг 3. Скопировать URL и ключ (ты)

1. В левом меню: **Project Settings** (иконка шестерёнки внизу) → вкладка **API**.
2. Скопируй:
   - **Project URL** (например `https://xxxxx.supabase.co`);
   - **anon public** key (длинная строка `eyJ...`).
3. В корне проекта создай или отредактируй **`.env.local`** (он не коммитится в git) и добавь:
   ```
   NEXT_PUBLIC_SUPABASE_URL=https://твой-project-url.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=твой_anon_key
   ```
   Подставь свои значения без кавычек.

После этого напиши, что шаги 1–3 сделаны — подключу клиент Supabase в коде, формы входа/регистрации и работу с коллекцией (добавить/убрать монету, портфолио по списку из Supabase).
