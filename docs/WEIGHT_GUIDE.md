# Руководство по весам (граммы ↔ тройские унции)

Этот файл фиксирует, какие стандартные веса мы используем в проекте, как они выглядят в БД (`weight_g`, `weight_oz`) и что показываем пользователю.

## 1. Принципы

- Для драгметаллов используем **тройские унции**: 1 ozt = 31,1035 г.
- Для **килограммовых** монет в интерфейсе показываем **килограммы, граммы и унции** (унции считаем по формуле граммы / 31,1035, см. таблицу).
- Для дробных унций (1/2, 1/4, 1/10 и т.д.) используем веса из таблиц ЦБ и Perth Mint (округление по стандарту до двух знаков после запятой).

> Примечание: числа в столбце «Унции (ozt)» для килограммовых монет используются и в интерфейсе — в блоке «Вес в унциях» на странице монеты.

Числа для килограммов в столбце «Унции (ozt)» рассчитаны по формуле:

\[
\text{унции} = \frac{\text{граммы}}{31{,}1035}
\]

и округлены до двух знаков после запятой (для справок).

## 3. Как это связано с кодом

- **Форматирование характеристик и фильтров**:
  - `scripts/export-coins-to-json.js` и `lib/coinApiShape.ts` используют:
    - массив `WEIGHT_LABELS` — подбирает красивую подпись `weightLabel` по граммам;
    - словарь `WEIGHT_OZ_TO_LABEL` — когда в БД есть строка `weight_oz` (`"1/4 унции"`, `"1 кг"` и т.п.).
- **Килограммовые монеты**:
  - В `WEIGHT_LABELS` есть 1 кг, 2 кг, 3 кг, 5 кг, 10 кг — для них `weightLabel` вида «1 кг · 1000 грамм» и т.п.
  - В блоке «Вес в унциях» на странице монеты для кг показываем рассчитанные унции (32,15 унции для 1 кг, 64,30 для 2 кг и т.д.) через `weightOzDisplay`.

## 4. Рекомендации для скриптов

- При проверке согласованности весов:
  - использовать **тройские унции** (1 ozt = 31,1035 г);
  - для дробных унций сверять `weight_g` с таблицей выше;
  - для килограммов проверять только граммы (унции считать вспомогательно, но не выводить пользователю).
- При заполнении `weight_oz` в БД:
  - для унций использовать строки из столбца «Значение `weight_оз` в БД»;
  - для килограммов — `1 кг`, `3 кг`, `5 кг` (или оставлять `NULL`, если в интерфейсе достаточно `weight_g` и `weightLabel`).
