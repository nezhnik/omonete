# Экспорт коллекции в Excel — что нужно и как сделать

Документ описывает возможность скачивания коллекции пользователя в Excel. Реализация отложена.

---

## 1. Что у нас уже есть

- **Кнопка «Скачать Excel»** на странице «Моя коллекция» — уже отображается, но без логики.
- **Данные в памяти:** `portfolioRows` — массив `PortfolioRow` с полями: id, title, series, mintName, mintCountry, faceValue, metalLabel, mintage, buyPrice, coinPrice, metalPrice, weightG и др.
- **Источник:** `collectionIds` из Supabase + данные монет из `/data/coins/[id].json`.

---

## 2. Чем экспорт проще импорта

| Аспект | Импорт | Экспорт |
|--------|--------|---------|
| Источник данных | Чужой файл | Уже загружено в браузере (portfolioRows) |
| Matching | Нужен mapping к нашим id | Не нужен — данные наши |
| Сервер | Может понадобиться | Не нужен |
| БД | Много insert | Нет обращений к БД |
| Лимиты Supabase | Да | Нет |

Экспорт можно сделать **полностью на клиенте** — генерация Excel в браузере и скачивание файла.

---

## 3. Что нужно сделать

### 3.1. Библиотека для Excel

Использовать одну из:

- **xlsx (SheetJS)** — популярная, `npm install xlsx`, создание и скачивание в 10–20 строк.
- **exceljs** — больше возможностей (стили, несколько листов), чуть тяжелее.

Рекомендация: **xlsx** — достаточно для таблицы с коллекцией.

### 3.2. Формирование таблицы

Собрать массив строк из `portfolioRows` (или `filteredRows`, если действует поиск):

```ts
// Пример структуры строки для Excel
[
  ["ID", "Название", "Серия", "Монетный двор", "Страна", "Номинал", "Металл", "Тираж", "Количество", "Цена покупки, ₽", "Вес, гр."],
  ["3637", "Троицкий собор...", "Эпоха просвещения", "ЛМД", "Россия", "3 рубля", "Серебро", "40 000", "1", "—", "31,1"],
  // ...
]
```

### 3.3. Вызов скачивания

1. `XLSX.utils.aoa_to_sheet(data)` — лист из массива.
2. `XLSX.utils.book_new()` + `book_append_sheet` — книга.
3. `XLSX.writeFile(workbook, "моя-коллекция.xlsx")` — скачивание.

### 3.4. Колонки для экспорта

Предлагаемые колонки (совместимо с импортом и с таблицей на экране):

| Колонка | Источник | Зачем |
|---------|----------|-------|
| coin_id | row.id | Для повторного импорта |
| Название | row.title | Удобство |
| Серия | row.series | |
| Монетный двор | row.mintName | |
| Страна | row.mintCountry | |
| Номинал | row.faceValue | |
| Металл | row.metalLabel | |
| Тираж | row.mintage | |
| Количество | collectionCounts[i] | Если есть |
| Цена покупки, ₽ | row.buyPrice или из БД | Если храним |
| Вес, гр. | row.weightG | |

## 4. Пошаговый план реализации

1. **Установить зависимость:** `npm install xlsx` (или `@types/xlsx` если нужны типы).
2. **Функция экспорта:** `exportToExcel(rows: PortfolioRow[], counts?: number[])` — формирует массив, создаёт книгу, вызывает `writeFile`.
3. **Обработчик кнопки:** `onClick={() => exportToExcel(filteredRows, collectionCounts)}`.
4. **Имя файла:** `моя-коллекция-YYYY-MM-DD.xlsx` или просто `моя-коллекция.xlsx`.
5. **Пустая коллекция:** отключить кнопку или показать toast «Добавьте монеты для экспорта».

---

## 5. Нюансы

- **Поиск:** экспортировать `filteredRows` (то, что видно после фильтра) или всегда `portfolioRows`? Логичнее — текущий вид (filteredRows).
- **Количество и цена покупки:** пока хранятся локально (`collectionCounts`, mock в `buyPrice`). Если появятся в БД — подтянуть в `PortfolioRow` и включить в экспорт.
- **Размер:** 500–1000 строк — нормально для xlsx в браузере, тормозов не будет.
- **Статика:** библиотека xlsx добавляет ~200–400 КБ в бандл. При необходимости — dynamic import: `const XLSX = await import('xlsx')`.

---

## 6. Связь с импортом

Экспорт даёт файл с колонкой `coin_id`. Пользователь может:

- Редактировать в Excel и загрузить обратно (импорт по `coin_id`).
- Делать бэкап коллекции.
- Переносить данные в другие системы.

Формат экспорта желательно сделать таким же, как шаблон для импорта: те же колонки, тот же порядок.

---

## 7. Минимальная реализация

1. `npm install xlsx`
2. В `portfolio/page.tsx`: `handleExportExcel` → формирование массива из `filteredRows` → `XLSX.writeFile`.
3. Подключить `onClick` к кнопке «Скачать Excel».
4. Обработка пустой коллекции: `if (filteredRows.length === 0) return` + toast.

Оценка трудозатрат: 1–2 часа.
